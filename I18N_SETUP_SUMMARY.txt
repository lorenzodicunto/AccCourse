================================================================================
QUICK START
================================================================================

1. Import and use in client components:

   import { useI18n } from '@/lib/i18n';

   function MyComponent() {
     const { t, locale, setLocale } = useI18n();
     return <h1>{t('dashboard.title')}</h1>;
   }

2. Add language switcher to header/navbar:

   import { LanguageSwitcher } from '@/components/LanguageSwitcher';

   <LanguageSwitcher />

3. Access full translations object:

   const { t, translations } = useI18n();
   const title = translations.dashboard.title;

4. Programmatically change language:

   const { setLocale } = useI18n();
   setLocale('en');
   setLocale('es');

================================================================================
ADDING NEW TRANSLATIONS
================================================================================

To add a new translation key:

1. Update I18nStrings interface in src/lib/i18n.ts:
   notifications: {
     saved: string;
   };

2. Add to all 4 language objects (ptBR, en, es, fr):
   notifications: {
     saved: "Salvo com sucesso!",
     // ... other languages
   };

3. Use in components:
   const { t } = useI18n();
   const message = t('notifications.saved');

================================================================================
NOTES
================================================================================

- Default language: PT-BR
- Language preference is saved to localStorage as 'i18n-locale'
- All translations are bundled at build time (no runtime loading)
- The system is purely client-side; no API calls required
- I18nProvider must wrap the entire app for context to work
- Components using useI18n() must be marked with 'use client'
